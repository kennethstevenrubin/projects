Now that it is done it is embarrasingly complex.

There are a few things I would do differently if I thought we were going with this.
For instance, I would color the block area a different color than the expression areas.
I would move the scroll bars out from the parameter and statement sections of the method to scroll the whole method and I would have the method parameter section grow dynamically, like the invocation statement formal parameter list.
I would move the "add" rows for statements and parameters out of their respective areas.
Well, frankly, there would be a bunch of changes--but this is the gist of the design I imagined.
Also, this completely leaves out the complexity of expressions and expression building.
And the add functionality is non functioning, but you can scroll with the scroll glyphs and inside the statement block.

Unfortunately for me, this model is surprisingly complete--from an architectural perspective.
For instance, the for statement could contain another for statement and everything would properly nest and such.
And the blocks grow and size as they should. So if you could add new statements they would merge into the structure perfectly.
Plus there's plenty of general architecture too, like: Point, and Size and Area and Triangle classes. Though I don't know what use they will get now.

Anyway, this mess basically is the sample method:
function LoopAndAlert(iMaxCount) { for (var i = 0; i < iMaxCount; i++) { alert(i); } }
